# Implementation Cuts

This file exists to protect the demo.

## Critical Path

Build these in order:

1. Seed data script.
2. Redis leaderboard.
3. Minimal UI.
4. Vapi intelligence answer from Redis.
5. TinyFish visible scan adapter.
6. Publish log.
7. Shipables skill.

That order is intentional. A seeded Redis leaderboard plus Vapi voice can already demo the core value while TinyFish is being stabilized.

## Cut Order

Cut first:

1. LinkedIn live crawl.
2. WebSockets.
3. Ghost/TigerData persistence.
4. real x402 payment rail.
5. dynamic Vapi persona assistant creation.
6. second voice assistant.
7. cross-platform dedupe.
8. full cited.md live publish if credentials/API are slow.

Do not cut:

- seed data.
- Redis sorted set leaderboard.
- visible TinyFish activity.
- Vapi Mode A intelligence answer.
- demo script rehearsal.

## MVP vs Stretch

| Feature | MVP? | Notes |
|---|---:|---|
| Seeded fan profiles | Yes | Required demo insurance. |
| Redis sorted set ranking | Yes | Best Redis track story. |
| TinyFish live stream | Yes | Can be one controlled scan if social pages fail. |
| Vapi top fan answer | Yes | Simplest Vapi track story. |
| Frontend leaderboard | Yes | Needed for visual demo. |
| Shipables skill | Yes | Required by submission strategy. |
| cited.md publish log | Yes | Can be stubbed if live API is slow. |
| Ghost/TigerData | No | Good sponsor depth, but too risky as hard dependency. |
| LinkedIn | No | Adds fragility. |
| Fan persona voice mode | Stretch | Great wow moment, but must not block Mode A. |
| x402 real payments | Stretch | Mention only if real. |

## Decision Rules

At 1:00 PM:

- If TinyFish social crawl is not producing useful fan data, switch to seeded data for main demo and use TinyFish as visible research/agent trace.

At 2:00 PM:

- If Vapi custom LLM is not stable, switch to Vapi custom tool or static answer endpoint.

At 3:00 PM:

- Stop adding integrations. Polish UI and rehearse.

At 4:00 PM:

- Commit, publish skill if possible, and prepare submission.

## Language To Use If A Fallback Is Active

Use:

> The live scan is still running, so I am switching to a saved run generated by the same data schema to keep the 3-minute demo tight.

Avoid:

> This part is mocked.

Use:

> The payment rail is shown as a publish event in this demo; the adapter is isolated so the live cited.md integration can be swapped in.

Avoid:

> We did not get cited.md working.
